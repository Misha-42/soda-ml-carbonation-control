````md
## Зачем это нужно

Процесс карбонизации аммонизированного рассола — центральная стадия
аммиачно-содового процесса (метод Сольве).
На практике лабораторный контроль выполняется раз в смену, и если что-то пошло не так,
оператор узнаёт об этом с задержкой 4–8 часов.
За это время возможны:
- рост брака,
- перерасход реагентов,
- дополнительная нагрузка на оборудование.

ML-модель решает эту проблему: она обучается на исторических данных процесса
и выдаёт прогноз до того, как отклонение успевает накопиться.

```mermaid

    A([Параметры процесса]) --> B([ML soft sensor])

    C --> D([Оператор реагирует заранее])
    D --> E([Снижение брака и потерь])

    style A fill:#dbeafe,stroke:#2563eb,color:#1a1a1a
    style B fill:#fef9c3,stroke:#ca8a04,color:#1a1a1a
````

## Что исследуется

В рамках НИР-2 строится базовая ML-система для прогноза трёх сменных показателей карбонизации.

| Таргет       | Описание                         | Сложность прогноза | Приоритет |
| ------------ | -------------------------------- | ------------------ | --------- |
| `target_t`   | Температура суспензии NaHCO₃, °C | Низкая             | №1        |
| `target_cl`  | Содержание ионов Cl⁻, г/л        | Средняя            | №2        |
| `target_nh3` | Свободный NH₃, г/л               | Высокая            | №3        |

Основная гипотеза:
нелинейные модели (`Random Forest`, `XGBoost`, `Gradient Boosting`) дадут преимущество
над линейной моделью там, где технологические зависимости выражены нелинейно,
а технологически осмысленный отбор признаков будет устойчивее
полностью автоматического отбора на малой выборке.

## Результаты

На текущем этапе репозиторий хранит не один результат, а набор результатов по нескольким веткам исследований.

Ключевые выводы по baseline-направлению:

* для `target_t` наилучшие результаты показывают ансамблевые методы;
* для `target_cl` линейная модель в ряде постановок остаётся конкурентной;
* `target_nh3` остаётся самым трудным таргетом и требует расширения признаков и более сильной инженерии данных.

Ключевой вывод по структуре исследований:

* нет одной универсально лучшей модели для всех технологических показателей;
* подход должен быть **таргет-специфичным**;
* рост качества сильнее зависит от качества признаков и постановки задачи,
  чем от простой замены одного алгоритма другим.

## Последняя и самая информативная версия: soda-ml-nir_targetB_v6

### Что это

[`soda-ml-nir_targetB_v6`](experiments/soda-ml-nir_targetB_v6/) — последняя и самая информативная версия проекта на текущем этапе.

Главная ценность этой версии — не только в абсолютных метриках, а в том, что здесь
экспериментальная схема стала заметно более строгой, воспроизводимой и управляемой.

В этой версии:

* жёстче контролируется утечка информации;
* baseline фиксируется как воспроизводимая конфигурация;
* список baseline-признаков сохраняется отдельным файлом;
* результат дополнительно проверяется через временную валидацию;
* различаются понятия «лучший результат на одном split» и «рабочий baseline».

### Текущая постановка задачи в v6

По материалам проекта `v6` исследование выполнено для задачи прогнозирования `target1`
на основе агрегированных признаков процесса.

В качестве основного датасета использовался:

* `dataset_target1_baseline_v1.csv`

Ключевые характеристики постановки:

* **98 наблюдений**
* временная колонка: `target_timestamp_for_scada`
* целевая переменная: `target_value`
* временное разбиение **80/20** без перемешивания
* базовые модели:

  * `Ridge`
  * `RandomForestRegressor`
  * `GradientBoostingRegressor`

### Логика экспериментальной серии v6

Экспериментальная серия была выстроена как последовательность шагов:

1. базовый запуск на полном наборе признаков `w60__ + w120_30__`;
2. переход к схеме `w60_only`;
3. ограниченный light tuning для `Random Forest`;
4. ручное сокращение признаков до `mean + last`;
5. автоматический importance-based отбор признаков;
6. фиксация лучшего baseline-набора;
7. отдельная проверка устойчивости через `TimeSeriesSplit(4)`.

### Главный результат v6

Лучшим рабочим baseline в `v6` стал вариант:

**Random Forest + w60_only + top_100 features by train importance**

По сравнению с baseline уровня `exp_02` удалось:

* сократить число признаков с **769 до 100**;
* улучшить `RMSE` с **4.116668** до **3.996800**;
* повысить `R²` с **0.113823** до **0.164679**.

### Почему эта версия важна

Версия `v6` важна прежде всего как:

* наиболее аккуратно оформленный baseline-контур;
* воспроизводимая экспериментальная схема;
* опорная версия для следующих этапов НИР;
* более зрелая методологическая постановка по сравнению с предыдущими версиями.

Набор команд зависит от конкретного проекта внутри `experiments/`.
Поэтому запуск лучше смотреть в README нужной папки.

Общая логика:

```bash
pip install -r requirements.txt
```

Далее перейти в нужный эксперимент и запускать его код согласно локальному README.

Примеры:

* [`experiments/rf_tuning_v5/README.md`](experiments/rf_tuning_v5/README.md)
* [`experiments/soda-ml-nir_targetB_v6/README.md`](experiments/soda-ml-nir_targetB_v6/README.md)

# soda-ml-nir_baseline_v1

Первый baseline-проект НИР по анализу технологического процесса производства соды.

Это самый ранний и самый простой этап исследования. Его задача — не построить окончательную промышленную модель, а быстро и прозрачно проверить, можно ли получить рабочий краткосрочный прогноз технологического показателя на основе данных SCADA.

На этом этапе сравниваются две базовые модели:
- `RandomForestRegressor`
- `XGBRegressor`

## Цель проекта

На первом этапе исследования ставилась задача построить **воспроизводимый baseline-контур** машинного обучения для краткосрочного прогноза технологического показателя процесса карбонизации по аппарату Кл-1.

В качестве целевой переменной использовалось значение температуры отходящего газа `temp_offgas_k1` на следующий шаг времени:
- горизонт прогноза: **t + 6 минут**
- тип задачи: **регрессия**
- источник данных: **SCADA**
- шаг дискретизации: **6 минут**

Главная цель baseline-этапа:
1. проверить, решаема ли задача прогноза в принципе;
2. получить первую воспроизводимую модель;
3. определить, какая из двух базовых моделей лучше подходит как отправная точка для следующих версий проекта.

---

## Что именно проверялось

В baseline-проекте сравнивались четыре конфигурации:

- `rf_small`
- `rf_medium`
- `xgb_small`
- `xgb_medium`

Качество оценивалось по трём метрикам:
- **MAE** — средняя абсолютная ошибка,
- **RMSE** — корень из среднеквадратичной ошибки,
- **R²** — коэффициент детерминации.

Разделение выборки выполнялось **по времени** (*time-based split*), без случайного перемешивания. Это важно для технологических временных рядов и позволяет избежать некорректной утечки информации из будущего.

## Краткий итог baseline-этапа

Лучшую точность в baseline-серии показала модель **RandomForestRegressor (`rf_medium`)**.

Итоговые метрики лучшей конфигурации:
- **MAE = 0.6245**
- **RMSE = 0.9311**
- **R² = 0.8904**

Это означает, что уже на самом первом этапе удалось получить сильный рабочий baseline для краткосрочного прогноза.

Главный вывод baseline-этапа:
- обе конфигурации `RandomForestRegressor` оказались лучше обеих конфигураций `XGBRegressor`;
- конфигурация `rf_medium` стала **рабочим baseline** для следующих версий проекта.

## Интерпретация результата

Для самого первого опыта результат можно считать удачным. Высокое значение **R² = 0.8904** показывает, что baseline-модель объясняет существенную часть вариации целевого сигнала на тестовой выборке.

Однако важно понимать ограничение этого результата: модель в значительной степени опирается на текущий признак `temp_offgas_k1`, который сам по себе очень близок к целевой переменной на следующем шаге. То есть baseline здесь во многом использует сильную краткосрочную автокорреляцию сигнала.

Это не делает результат плохим — наоборот, для первого этапа это полезно. Но это означает, что baseline нужно воспринимать как **отправную точку**, а не как окончательное решение.

## Структура проекта

- `data/` — входные CSV-данные для обучения.
- `src/data_prep.py` — загрузка CSV, базовая очистка, проверка обязательных столбцов, time-based split.
- `src/features.py` — минимальная подготовка признаков (числовые колонки + обработка пропусков).
- `src/train_baseline.py` — запуск baseline-экспериментов для RandomForest и XGBoost.
- `src/evaluate.py` — расчёт метрик и сохранение результатов/отчёта.
- `models/` — сохранённые модели по экспериментам.
- `reports/` — CSV с метриками, сводка экспериментов, Markdown-отчёт и графики.
- `nir/` — текстовые главы НИР по этому baseline-проекту.

## Быстрый запуск baseline

### 1. Установите зависимости

```bash
pip install -r requirements.txt
````

### 2. Положите CSV в папку `data/`

CSV должен содержать:

* целевую колонку (например, `target`);
* временную колонку (например, `timestamp`) — если она есть, будет использован time-based split.

### 3. Запустите baseline-эксперименты

```bash
python src/train_baseline.py --data-path data/your_data.csv --target target --time-column timestamp
```

Если временной колонки нет, можно не передавать `--time-column`; тогда будет использован split по порядку строк.

## Ограничения baseline

Несмотря на хороший результат, текущий baseline имеет ряд ограничений:

1. это **самый первый опыт**, а значит модель решает задачу в базовой постановке;
2. существенная часть качества может быть связана с автокорреляцией целевого сигнала;
3. в проекте пока не используются:

   * лаговые признаки,
   * скользящие статистики,
   * специальные проверки на вклад отдельных групп признаков,
   * альтернативные таргеты.

Именно поэтому baseline следует рассматривать как первую рабочую контрольную точку проекта, а не как окончательную модель.

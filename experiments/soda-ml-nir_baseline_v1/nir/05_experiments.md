## 5.1. Постановка baseline-эксперимента

На первом этапе исследования был проведён baseline-эксперимент по краткосрочному прогнозированию технологического показателя процесса карбонизации на основе данных SCADA по аппарату Кл-1. В качестве исходного массива использовался временной ряд с шагом дискретизации 6 минут.

В качестве целевой переменной использовался показатель `target`, сформированный как значение температуры отходящего газа `temp_offgas_k1` на следующий шаг времени (t+6 минут). Такой подход позволил перейти от оценки текущего состояния к задаче краткосрочного прогноза.

В набор признаков были включены текущие значения технологических параметров процесса, в том числе:
- `temp_slurry_k1`,
- `gas_flow_in1_k1`,
- `gas_flow_in2_k1`,
- `level_k1`,
- `temp_offgas_k1`,
- `temp_out_water_k1`,
- `temp_zone_14_k1`,
- `temp_zone_5_k1`,
- `temp_zone_25_k1`,
- `gas_header_pressure_1`,
- `gas_header_pressure_2`,
- `fg_header_pressure`,
- `group_liquid_temp_g1`.

Разделение выборки на обучающую и тестовую выполнялось по времени (`time-based split`), что соответствует природе технологического процесса и исключает некорректное случайное перемешивание наблюдений.

Для сравнения были выбраны две модели машинного обучения:
- `RandomForestRegressor`,
- `XGBRegressor`.

Для каждой модели были протестированы по две конфигурации гиперпараметров:
- `rf_small`,
- `rf_medium`,
- `xgb_small`,
- `xgb_medium`.

Качество моделей оценивалось по следующим метрикам:
- MAE — средняя абсолютная ошибка,
- RMSE — корень из средней квадратичной ошибки,
- R² — коэффициент детерминации.

Дополнительно для каждой конфигурации сохранялись:
- обученная модель,
- таблица метрик,
- сводная таблица экспериментов,
- краткий текстовый отчёт,
- графики сравнения моделей,
- графики важности признаков.

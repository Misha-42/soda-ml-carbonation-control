# 06. Результаты

## 6.1. Результаты исходного baseline-сравнения моделей

На первом этапе исследования было выполнено baseline-сравнение трёх моделей: `Ridge`, `RandomForestRegressor` и `GradientBoostingRegressor` на полном наборе агрегированных признаков `w60__ + w120_30__`. Результаты показали, что линейная модель `Ridge` для рассматриваемой постановки не подходит: её ошибка оказалась слишком высокой, а коэффициент детерминации — резко отрицательным.

Наилучший результат внутри первого baseline-эксперимента продемонстрировал `Random Forest`, для которого были получены:
- MAE = 3.347818;
- RMSE = 4.150467;
- R² = 0.099212.

Для `Gradient Boosting` были получены немного лучшие значения MAE, однако по `RMSE` и `R²` эта модель уступала `Random Forest`. Поскольку для текущей задачи основными ориентирующими метриками считались именно `RMSE` и `R²`, в качестве рабочего baseline-кандидата был выбран `Random Forest`.

Этот результат показал, что в данных присутствует нелинейный прогнозный сигнал, но использование полного пространства из 1538 признаков при 98 наблюдениях выглядит чрезмерным. Поэтому следующий этап был направлен на сокращение признаков без потери качества.

---

## 6.2. Результаты перехода к схеме `w60_only`

Во втором эксперименте из исходного пространства были исключены все признаки `w120_30__`, а в модели оставлены только признаки `w60__`. Число признаков сократилось с 1538 до 769.

Несмотря на столь существенное упрощение, `Random Forest` не только не ухудшился, но и показал небольшое улучшение относительно исходного baseline:
- MAE = 3.365045;
- RMSE = 4.116668;
- R² = 0.113823.

Полученный результат имеет принципиальное значение. Он показывает, что более широкий набор агрегированных признаков не давал модели дополнительного полезного сигнала, а, напротив, создавал избыточность и повышал риск переобучения. Таким образом, схема `w60_only` оказалась более удачной и стала новым baseline для следующих этапов серии.

---

## 6.3. Результаты light tuning для Random Forest

После перехода к `w60_only` была выполнена ограниченная настройка `Random Forest`. Однако эксперимент показал, что простое изменение гиперпараметров не приводит к улучшению результата.

Для конфигурации с увеличенным числом деревьев и дополнительными ограничениями были получены:
- MAE = 3.397348;
- RMSE = 4.217209;
- R² = 0.070008.

По сравнению с `Experiment 2` качество ухудшилось. Это позволяет сделать вывод, что в текущей постановке основной резерв улучшения заключается не в тюнинге модели, а в сокращении и более разумной организации признакового пространства. Иными словами, на малой выборке попытка “дожать” качество за счёт гиперпараметров оказалась менее продуктивной, чем работа со структурой признаков.

---

## 6.4. Результаты ручного упрощения признаков: `mean + last only`

В четвёртом эксперименте было выполнено жёсткое ручное сокращение пространства `w60__`: в модели оставлялись только признаки типов `mean` и `last`, а признаки `std`, `min`, `max`, `delta` и вся группа `w120_30__` исключались.

Число признаков сократилось до 256. Для `Random Forest` были получены:
- MAE = 3.245783;
- RMSE = 4.156987;
- R² = 0.096379.

Несмотря на то что качество оказалось немного хуже, чем у `Experiment 2`, просадка была сравнительно небольшой. Это подтверждает наличие значительной избыточности в исходном пространстве признаков. Однако новый лучший baseline на данном этапе получен не был. Поэтому `Experiment 4` был сохранён как полезный компактный и интерпретируемый резервный вариант, но не как основной baseline.

---

## 6.5. Результаты importance-based отбора признаков

Наиболее важный результат серии был получен в `Experiment 5`, где использовался автоматический отбор признаков по `feature importance`, рассчитанной на обучающей части выборки. Проверялись четыре варианта:
- полный набор `w60__`;
- `top_30`;
- `top_50`;
- `top_100`.

Для полного набора `w60__` были получены:
- MAE = 3.394114;
- RMSE = 4.188671;
- R² = 0.082552.

Для подмножеств признаков были получены следующие результаты:
- `top_30`: MAE = 3.332403, RMSE = 4.129147, R² = 0.108442;
- `top_50`: MAE = 3.270822, RMSE = 4.056952, R² = 0.139346;
- `top_100`: MAE = 3.229928, RMSE = 3.996800, R² = 0.164679.

Лучший результат дал вариант `top_100`. Он оказался лучше как полного пространства `w60__`, так и baseline из `Experiment 2`. По сравнению с `exp_02` были получены:
- снижение RMSE с 4.116668 до 3.996800;
- рост R² с 0.113823 до 0.164679;
- сокращение числа признаков с 769 до 100.

Таким образом, importance-based selection внутри `w60_only` оказалась более эффективной, чем как использование полного набора признаков, так и ручное упрощение пространства по агрегатам. Именно этот результат позволил зафиксировать новый рабочий baseline для `target1`:  
**Random Forest + w60_only + top_100 features by train importance**.

---

## 6.6. Проверка устойчивости baseline через временные фолды

После выбора `top_100` как нового baseline была выполнена проверка устойчивости результата на нескольких временных разбиениях с использованием `TimeSeriesSplit(n_splits=4)`.

На отдельных фолдах были получены следующие результаты:
- `fold_1`: MAE = 2.122904, RMSE = 2.781229, R² = 0.019812;
- `fold_2`: MAE = 4.033000, RMSE = 5.172466, R² = 0.216860;
- `fold_3`: MAE = 4.291105, RMSE = 4.755381, R² = 0.166539;
- `fold_4`: MAE = 3.160833, RMSE = 3.920155, R² = 0.181504.

Сводные показатели по четырём фолдам составили:
- mean RMSE = 4.157308;
- std RMSE = 0.922905;
- mean R² = 0.146179;
- std R² = 0.076943.

Следует отметить, что первый фолд формировался на особенно маленькой обучающей подвыборке (`train = 22`), поэтому ожидать от него высокой стабильности было трудно. Тем не менее даже в этом случае коэффициент `R²` не ушёл в отрицательную область. На остальных фолдах качество также оставалось рабочим.

Это позволяет сделать аккуратный, но важный вывод: baseline `top_100` не является случайным удачным результатом только на одном split, а демонстрирует приемлемую устойчивость на нескольких временных срезах. В рамках текущего объёма данных такой результат достаточен для того, чтобы считать baseline рабочим и переносимым на следующий этап исследования.

## 6.8. Сравнение версии v6 с предыдущей версией проекта

Важно отметить, что текущую версию v6 нельзя напрямую сравнивать с предыдущей v5 по абсолютным значениям `RMSE` и `R²`, поскольку это разные постановки задачи, разные target-переменные и иной масштаб сложности. В предыдущем проекте лучший результат достигался для другой целевой переменной и другой задачи прогнозирования, поэтому более высокое значение `R²` в v5 не означает автоматически, что версия v6 слабее как исследовательская работа.

Если сравнивать проекты по качеству исследовательской постановки, то версия v6 заметно продвинулась вперёд. В частности:
- жёстче контролируется leakage;
- baseline фиксируется как воспроизводимая конфигурация;
- список baseline-признаков сохраняется отдельным файлом;
- результат дополнительно проверяется на устойчивость через walk-forward;
- различается понятие “лучший результат на одном split” и “рабочий baseline”.

Таким образом, по абсолютным метрикам качества текущий проект пока не превосходит лучший результат предыдущей версии, однако по методической строгости, воспроизводимости и зрелости экспериментальной схемы версия v6 является более сильной.

## 6.9. Итог по экспериментальной части

Проведённая серия экспериментов показала, что для задачи `target1` на текущем этапе наилучшим рабочим baseline является модель `Random Forest`, использующая только пространство признаков `w60_only` и фиксированный набор из 100 признаков, отобранных по `feature importance` на обучающей выборке.

По сравнению с предыдущим baseline (`exp_02`) удалось одновременно:
- сократить число признаков с 769 до 100;
- улучшить RMSE с 4.116668 до 3.996800;
- повысить R² с 0.113823 до 0.164679.

Дополнительная временная проверка через `TimeSeriesSplit` показала, что baseline сохраняет рабочее качество на нескольких временных фолдах и не демонстрирует системного развала по `R²`. Это позволяет считать его не просто лучшим результатом на одном разбиении, а рабоче устойчивым baseline в рамках текущего объёма данных.

Итогом данного этапа является не окончательная оптимальная модель “вообще”, а корректно построенный, воспроизводимый и достаточно устойчивый baseline-контур, который может использоваться как опорная конфигурация для следующего этапа НИР.

## 6.10. Ограничения текущей серии экспериментов

Несмотря на полученный результат, текущая серия имеет ряд ограничений.

Во-первых, объём выборки остаётся малым: в распоряжении имеется только 98 наблюдений. Это ограничивает статистическую устойчивость выводов и не позволяет делать слишком сильные утверждения о превосходстве одной конфигурации над другой.

Во-вторых, исследование выполнено для одного контура (`target1`) и одного фиксированного датасета. Следовательно, перенос выводов на другие target-переменные и другие производственные сценарии требует дополнительной проверки.

В-третьих, walk-forward валидация выполнялась для уже зафиксированного набора `top_100`, а не для полного pipeline с переотбором признаков на каждом фолде. С инженерной точки зрения это оправдано как проверка устойчивости baseline, однако формально это не эквивалентно повторной полной процедуре feature selection внутри каждого фолда.

В-четвёртых, качество модели пока оценивается только в статистических метриках MAE, RMSE и R². Для дальнейшего продвижения к практическому применению требуется интерпретация ошибки в терминах технологической допустимости и потенциальной полезности для поддержки управления процессом.

## Итоги текущего этапа проекта

На текущем этапе проекта разработан и протестирован воспроизводимый baseline-контур машинного обучения для `target1`. Была проведена последовательная серия экспериментов по сокращению и отбору признаков, по итогам которой лучшим решением стал вариант `Random Forest + w60_only + top_100 features by train importance`.

Дополнительно baseline был зафиксирован как конкретный набор признаков и проверен на устойчивость через временную валидацию. Полученный результат позволяет считать выбранную конфигурацию основой для дальнейшего переноса логики на следующий контур и для последующих этапов НИР, где акцент уже может быть смещён с поиска baseline на развитие признаковой инженерии, интерпретацию модели и расширение производственного применения.

СЛЕДУЮЩИЙ ШАГ: ПРОВЕРКА УСТОЙЧИВОСТИ BASELINE

Цель:
Проверить, что baseline exp_05 top_100 устойчив не только на одном time-based split,
но и на нескольких последовательных временных фолдах.

Что проверяем:
target1 / Random Forest / w60_only / top_100 by train importance

Файл с признаками:
outputs/exp_05_target1_topn_importance/baseline_feature_set_top100.csv

План проверки:
- использовать dataset:
  launch_target1_k1/dataset_target1_baseline_v1.csv
- загрузить baseline_feature_set_top100.csv
- оставить только признаки из этого списка
- сделать TimeSeriesSplit
- предпочтительно n_splits = 4
- если фолды получаются слишком маленькими, допустимо n_splits = 3

На каждом фолде считать:
- MAE
- RMSE
- R²

Что сохранить:
- метрики по каждому фолду
- среднее значение по всем фолдам
- standard deviation по всем фолдам

Критерий успеха:
Если:
- нет сильного развала метрик между фолдами
- средний RMSE остаётся близким к текущему уровню
- R² не уходит системно в отрицательную зону

тогда exp_05 top_100 можно считать рабочим устойчивым baseline.

Что НЕ делать сейчас:
- не запускать новую серию top_80 / top_120
- не менять модель
- не расширять сетку экспериментов

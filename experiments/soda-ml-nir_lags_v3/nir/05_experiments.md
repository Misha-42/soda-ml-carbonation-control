## 5.1. Постановка baseline-эксперимента

На первом этапе исследования был проведён baseline-эксперимент по краткосрочному прогнозированию технологического показателя процесса карбонизации на основе данных SCADA по аппарату Кл-1. В качестве исходного массива использовался временной ряд с шагом дискретизации 6 минут.

В качестве целевой переменной использовался показатель `target`, сформированный как значение температуры отходящего газа `temp_offgas_k1` на следующий шаг времени (t+6 минут). Такой подход позволил перейти от оценки текущего состояния к задаче краткосрочного прогноза.

В набор признаков были включены текущие значения технологических параметров процесса, в том числе:
- `temp_slurry_k1`,
- `gas_flow_in1_k1`,
- `gas_flow_in2_k1`,
- `level_k1`,
- `temp_offgas_k1`,
- `temp_out_water_k1`,
- `temp_zone_14_k1`,
- `temp_zone_5_k1`,
- `temp_zone_25_k1`,
- `gas_header_pressure_1`,
- `gas_header_pressure_2`,
- `fg_header_pressure`,
- `group_liquid_temp_g1`.

Разделение выборки на обучающую и тестовую выполнялось по времени (`time-based split`), что соответствует природе технологического процесса и исключает некорректное случайное перемешивание наблюдений.

Для сравнения были выбраны две модели машинного обучения:
- `RandomForestRegressor`,
- `XGBRegressor`.

Для каждой модели были протестированы по две конфигурации гиперпараметров:
- `rf_small`,
- `rf_medium`,
- `xgb_small`,
- `xgb_medium`.

Качество моделей оценивалось по следующим метрикам:
- MAE — средняя абсолютная ошибка,
- RMSE — корень из средней квадратичной ошибки,
- R² — коэффициент детерминации.

Дополнительно для каждой конфигурации сохранялись:
- обученная модель,
- таблица метрик,
- сводная таблица экспериментов,
- краткий текстовый отчёт,
- графики сравнения моделей,
- графики важности признаков.

## 5.2. Эксперимент по исключению признака `temp_offgas_k1`

После получения baseline-результатов был выполнен дополнительный эксперимент, направленный на оценку зависимости модели от доминирующего признака `temp_offgas_k1`.

Для этого были сформированы два варианта входного признакового пространства:
- полный набор признаков, включающий `temp_offgas_k1`;
- модифицированный набор признаков, из которого `temp_offgas_k1` был исключён.

Во всех остальных аспектах постановка эксперимента сохранялась неизменной:
- использовался тот же CSV-файл,
- применялось time-based split,
- сохранялся тот же target,
- использовались те же конфигурации RandomForestRegressor и XGBRegressor,
- сравнение выполнялось по метрикам MAE, RMSE и R².

Цель эксперимента состояла в том, чтобы оценить, насколько качество модели определяется текущим значением целевого сигнала и в какой степени остальные технологические параметры способны самостоятельно обеспечивать приемлемый уровень прогноза.

## 5.3. Эксперимент с лаговыми и скользящими признаками

После анализа baseline-результатов и оценки влияния доминирующего признака `temp_offgas_k1` был проведён следующий этап экспериментов, направленный на учёт временной динамики процесса.

Для этого в признаковое пространство были добавлены:
- лаговые признаки для отдельных технологических параметров;
- скользящие средние по короткому временному окну.

В качестве базовых параметров для построения временных признаков использовались:
- `temp_offgas_k1`,
- `gas_flow_in1_k1`,
- `gas_flow_in2_k1`,
- `gas_header_pressure_1`,
- `gas_header_pressure_2`,
- `fg_header_pressure`.

Для указанных признаков были построены лаги по предыдущим временным шагам, а также простые скользящие средние.

Во всех остальных аспектах постановка эксперимента сохранялась неизменной:
- использовался тот же исходный CSV-файл;
- применялось разбиение выборки по времени;
- сохранялся тот же target;
- сравнивались конфигурации `RandomForestRegressor` и `XGBRegressor`;
- оценка выполнялась по метрикам MAE, RMSE и R².

Цель эксперимента состояла в том, чтобы оценить, насколько использование истории изменения параметров процесса позволяет повысить точность краткосрочного прогноза.
